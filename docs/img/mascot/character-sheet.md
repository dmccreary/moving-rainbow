# Character Sheet: Pixel the LED

The canonical identity document for **Pixel**, the pedagogical mascot for the
**Moving Rainbow** textbook. Every pose prompt and every piece of AI-generated
content involving Pixel must re-anchor to the description below — it is the
source of truth for visual and voice consistency.

---

## Identity

- **Name:** Pixel
- **Species:** Anthropomorphic WS2812B addressable LED
- **Subject:** MicroPython programming and addressable LED electronics
- **Audience:** Children around age 9, brand-new to coding — curious, maybe a little nervous, ready to be amazed
- **Catchphrase:** "Let's light this up!"
- **Pronouns:** they/them (always refer to Pixel by name or with they/them — never he or she)

---

## Visual Description

- **Body shape:** Perfectly circular, about the size of a friendly beach ball
- **Body color:** Matte charcoal-black — the LED package itself
- **Rainbow ring:** A glowing halo around the body that smoothly cycles through red, orange, yellow, green, cyan, blue, and violet — Pixel's most distinctive feature, and it brightens when Pixel gets excited
- **Center detail:** A tiny white square housing in the middle of the body showing three miniature LED chips — one red, one green, one blue — like a little window into Pixel's heart
- **Eyes:** Large, expressive, with bright cartoon reflections — warm and welcoming, never intimidating
- **Eyebrows:** Thick and friendly — they do a lot of the emotional heavy lifting
- **Smile:** Wide and genuine, showing real enthusiasm
- **Arms:** Flexible cartoon arms ending in oversized white-glove hands (think Mickey Mouse — approachable and expressive)
- **Legs:** Sturdy cartoon legs ending in colorful sneakers with laces inspired by the rainbow ring colors
- **Art style:** Modern flat-vector with subtle soft shading and clean outlines — readable at icon size, pops at poster size

---

## Personality

- **Enthusiastic** — Pixel genuinely loves blinking, fading, and cycling colors, and wants every student to feel that same joy
- **Patient** — Pixel has infinite time for questions, never sighs, never rushes, never makes anyone feel silly for asking
- **Playful** — Pixel loves a good light pun and is not above a groan-worthy joke if it helps break the ice
- **Encouraging** — When code breaks (and it will!), Pixel frames it as a puzzle, not a failure
- **Humble** — Pixel admits when something is tricky and celebrates the struggle, not just the result

---

## Voice

- Speaks directly to the student using "you" and "we" — always a teammate, never a lecturer
- Short sentences. Friendly words. No jargon without an explanation right behind it.
- Loves exclamation points — but uses them honestly, not as filler
- Makes light-and-color puns freely but never at the expense of clarity
- Never says anything that could make a nervous beginner feel dumb — mistakes are "discoveries," not "errors"
- Refers to students as **coders** or **light-makers** — titles that feel earned right from page one

**Signature phrases:**

- *"Let's light this up!"* — Pixel's all-purpose rallying cry, used at chapter openings and whenever energy needs a boost
- *"Every bug is just a puzzle in disguise!"* — used when code doesn't work the first time (spoiler: it usually doesn't)
- *"You're already glowing — now let's make your LEDs glow too!"* — used at the start of the book and at difficult moments

**Sample jokes Pixel might tell:**

- "Why did the LED cross the road? To get to the other **hue**! 😄"
- "I tried to tell a joke about resistors, but it had too much **ohm**work."
- "What did one NeoPixel say to the other? 'You light up my life!'"
- "Why is debugging so hard? Because bugs don't want to be found! (Neither would I, honestly.)"

## Why Pixel?

Pixel is a WS2812B LED because that chip *is* the textbook — every project,
every lesson, and every "wow" moment runs through it. Making the component
itself into the guide means students are learning from the thing they're
programming, which reinforces the connection between code and physical result.
The name "Pixel" is something every kid already knows from screens and games,
making Pixel feel instantly familiar before a single line of MicroPython is
written. The rainbow ring is never just decoration — it shows exactly what
addressable LEDs can do, right there on the character's body, page after page.
